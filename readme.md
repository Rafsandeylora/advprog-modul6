## Commit 1 Reflection notes

Pada milestone ini saya belajar bahwa method `handle_connection` adalah bagian yang bertugas memproses koneksi yang datang dari browser setelah server menerima `TcpStream`. Sebelumnya program hanya mencetak pesan bahwa koneksi berhasil dibuat, tetapi setelah menambahkan method ini, server mulai membaca isi request HTTP yang dikirim oleh browser. Saya memahami bahwa `BufReader::new(&mut stream)` digunakan agar data dari stream dapat dibaca dengan lebih efisien dan lebih mudah, khususnya per baris. Method `.lines()` kemudian mengubah isi stream menjadi iterator baris-baris teks, sehingga header HTTP seperti `GET / HTTP/1.1`, `Host`, dan `User-Agent` dapat dilihat dengan jelas. Bagian `.map(|result| result.unwrap())` dipakai untuk mengambil nilai dari setiap hasil pembacaan baris, walaupun pendekatan ini masih sederhana karena program akan langsung panic jika terjadi error. Saya juga memahami bahwa `.take_while(|line| !line.is_empty())` sangat penting karena request HTTP memiliki baris kosong sebagai penanda akhir header, sehingga program berhenti membaca di titik yang tepat. Dari hasil percobaan ini saya jadi lebih mengerti bahwa browser sebenarnya mengirim request dalam format teks yang terstruktur, dan server perlu membacanya terlebih dahulu sebelum memutuskan response apa yang akan diberikan. Hal lain yang saya pelajari adalah isi request dapat berbeda-beda pada tiap mesin atau browser, sehingga output yang saya lihat belum tentu sama persis dengan contoh di modul. Secara keseluruhan, milestone ini membantu saya memahami alur dasar komunikasi antara browser dan web server, yaitu koneksi diterima, request dibaca, lalu nantinya akan diproses lebih lanjut pada milestone berikutnya.

## Commit 2 Reflection notes

Pada milestone ini saya memahami bahwa method `handle_connection` tidak lagi hanya membaca request dari browser, tetapi juga mulai membentuk response HTTP yang valid untuk dikirim kembali. Saya belajar bahwa browser membutuhkan format response tertentu agar dapat merender isi halaman dengan benar, sehingga server harus mengirim status line seperti `HTTP/1.1 200 OK` sebelum isi HTML. Saya juga memahami fungsi `fs::read_to_string("hello.html")`, yaitu membaca file HTML dari direktori project lalu menyimpannya sebagai string yang bisa dikirim ke browser. Bagian `Content-Length` ternyata penting karena browser perlu mengetahui panjang isi response agar dapat memproses body dengan benar. Saya baru benar-benar paham bahwa string `\r\n\r\n` berfungsi sebagai pemisah antara header HTTP dan body response, sehingga browser dapat membedakan metadata response dan isi halaman. Selain itu, saya menyadari bahwa file `hello.html` harus berada di lokasi yang benar dan `cargo run` harus dijalankan dari direktori project yang tepat, karena program membaca file tersebut dengan path relatif. Dari percobaan ini saya melihat bahwa web server sederhana sebenarnya hanya mengirim teks dalam format HTTP, lalu browser yang bertugas merendernya menjadi halaman visual. Milestone ini membuat saya lebih paham bahwa komunikasi client-server di web pada dasarnya cukup sederhana, yaitu browser mengirim request dan server membalas dengan response yang mengikuti protokol HTTP.
